# Research Protocol Draft

## Objective

Evaluate whether gamification improves learning outcomes, engagement, motivation, usability, and satisfaction in a microlearning-based coding application for busy beginner learners.

## Respondent Criteria

Inclusion criteria:

- Ordinary individuals with little to no coding experience.
- Busy with work, study, or daily responsibilities.
- Interested in learning coding.
- Not professional programmers.

Exclusion criteria:

- Intermediate or advanced programming knowledge.
- Professional software development experience.
- Prior completion of a formal programming course beyond beginner level.

## Sample Size

Total respondents: 30 people.

- 15 respondents: Gamification OFF.
- 15 respondents: Gamification ON.

## Experimental Design

Quasi-experimental pretest-posttest design with two groups.

| Group | Version | Description |
|---|---|---|
| Control | Gamification OFF | Microlearning coding app without XP, levels, badges, streaks, or rewards |
| Experimental | Gamification ON | Same app and content, with XP, levels, badges, streaks, and rewards |

## Procedure

1. Recruit respondents.
2. Assign respondent IDs and group codes.
3. Collect demographic data.
4. Collect basic coding background data.
5. Conduct pretest.
6. Let respondents use the assigned app version.
7. Collect module progress logs.
8. Conduct posttest.
9. Collect questionnaire responses.
10. Analyze learning gain, engagement, motivation, usability, satisfaction, and gamification element perception.

## Main Variables

| Variable Type | Variable | Measurement |
|---|---|---|
| Independent variable | Gamification status | ON vs OFF |
| Dependent variable | Learning outcome | Pretest, posttest, gain score |
| Dependent variable | Engagement | Questionnaire and activity logs |
| Dependent variable | Motivation | Questionnaire |
| Dependent variable | Usability | Questionnaire |
| Dependent variable | Satisfaction | Questionnaire |
| Controlled variable | Learning content | Same modules in both groups |
| Controlled variable | Learning structure | Same microlearning design in both groups |

## Data Analysis

Suggested analyses:

- Pretest vs posttest score comparison.
- Gain score calculation.
- Gamification ON vs OFF gain score comparison.
- Mean score calculation for questionnaire sections.
- Activity log analysis: completed modules, time spent, attempts, and quiz accuracy.

Suggested statistical tests:

- Paired-sample t-test if data is normal.
- Wilcoxon signed-rank test if within-group data is not normal.
- Independent-sample t-test if between-group data is normal.
- Mann-Whitney U test if between-group data is not normal.

## Pretest/Posttest Note

Pretest and posttest questions should be different but equivalent in difficulty and topic distribution.
